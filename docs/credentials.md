## 認証情報・環境変数の扱い

APIキー等の認証情報を環境変数で渡す場面では、**Claude Code がどのプラットフォームで動いているかによって環境変数の正しい保存場所が変わる**。スキル・セットアップ手順は、**自分の実行環境を認識したうえで、ユーザーに「あなたの環境ではここに入れてください」と案内する**こと。秘密の値は**リポジトリに置かない・エコーしない・コミットしない**（公開範囲ルールと直結。「このリポに置いたもの＝顧客に見えるもの」）。

### まず2系統を区別する（混同しない）

| 系統 | 用途 | 置き場所 | 動作環境依存 |
| --- | --- | --- | --- |
| **GitHub Actions Secrets** | 夜間 cron 等の**自動同期**（サーバー側で無人実行） | `gh secret set`（org/repo secret） | **しない**（GitHub側） |
| **対話実行用の環境変数** | 手元で `/backlog-push`・`/backlog-pull`・`/sync-designs` 等を**手動実行**する時 | 動作環境ごとに異なる（下表） | **する** |

以下はすべて**後者（対話実行用）**の話。

### 動作環境判定

- 環境変数 `CLAUDE_CODE_REMOTE` が `true` → **Web版（claude.ai/code・クラウド実行）**。
- それ以外 → **ローカル実行**（ターミナルCLI または デスクトップアプリ）。CLI/デスクトップの区別は判定しづらいので、ローカルなら両方の入れ方を案内してよい。

### 動作環境別の保存場所

| 動作環境 | 環境変数の入れ方 | 保護レベル |
| --- | --- | --- |
| **ターミナル(CLI)** | リポジトリルートの `.env`（gitignore済み）／シェルの `export`／`~/.claude/settings.json` の `env` | ローカル |
| **デスクトップアプリ** | プロンプト横の環境ドロップダウン → **Local** → 歯車（**ローカル環境エディタ**）。**マシンに暗号化保存**され全ローカルセッションに適用。または `~/.claude/settings.json` の `env`。※デスクトップはシェル環境を丸ごと継承しないため、`export` だけに頼らない | ローカル・**暗号化** |
| **Web（クラウド実行）** | 環境セレクタ → **Add/Edit environment** の環境変数欄に `.env` 形式（`KEY=value` を1行ずつ・**引用符なし**）。リポの `.env` はクローンに含まれない（gitignore）ので**届かない** | ⚠️ **専用シークレットストアは未提供**。環境設定の環境変数は**その環境を編集できる人に可視**（暗号化金庫ではない）。**共有環境に秘密を置かず、各自が自分の環境に持つ** |

### スキルの読み込み規約

スキルは**リポの `.env` ファイルだけに依存しない**。**環境変数が既にあればそれを使い、無ければ `.env` にフォールバック**する。これで CLI（`.env`）・デスクトップ（ローカル環境エディタ）・Web（クラウド環境設定）の3つの動作環境が同じスキルで動く。

```bash
# 環境変数優先・無ければ .env を読む（1Password fifo 対応で source ではなく eval）
set -a; [ -e .env ] && eval "$(grep -v '^#' .env)"; set +a
```

必須変数が揃っているかは読み込み後に検証し、欠けていたら**動作環境に応じた入れ場所**を案内する（例: Webなら「環境設定の環境変数欄に追加」、デスクトップなら「ローカル環境エディタに追加」）。
