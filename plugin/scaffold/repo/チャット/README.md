# チャット

Slack 等のチャットは**このリポジトリにミラーしません**。正本は常に Slack 側です（古いコピーを作らない／内部限定情報を顧客開放リポに持ち込まないため）。`開発/issues/` と同じ「道しるべ」方式です。

## 参照方法

対象チャンネルは `channels.json` に `{ "name": "...", "url": "..." }` の形で列挙し（`name` はどのチャンネルか分かるラベル）、Claude Code 等から **Slack MCP** でライブ取得します（Slack MCP の接続設定は cortex プラグインの `.mcp.json` 側）。

## 権限境界（公開範囲）

Slack MCP に接続できるのは**社内メンバーのみ**です。顧客はコネクタの権限を持たないため、ここからチャット内容を読むことはできません。**これが公開範囲の境界**になります。

> Slack を丸ごと貯めると内部限定の雑談・評価が漏れる。だから中身は置かず、読める人だけが MCP 経由でライブ参照する。

## セットアップ

対象 Slack チャンネルを `channels.json` に **`{ "name": "チャンネル名", "url": "リンク" }` の形で**列挙します（複数可。`name` で「どのチャンネルか」が分かるようにする）。`/setup-project` のステップ9でも案内されます。
