# このリポジトリの使い方ガイド

案件メンバー（エンジニア・非エンジニア問わず）が、このコンテキストリポジトリを**日々どう使うか**をまとめた実用ガイドです。「こういう時はこうする」「ここに何を置く」「どう作業する」を、シナリオ別に引けるようにしています。

> ドキュメントの役割分担（このガイドは「日々の使い方」担当）
>
> | ドキュメント | 役割 |
> | --- | --- |
> | **USAGE.md（このファイル）** | 日々の使い方・シナリオ別の手順 |
> | `README.md` | リポジトリの概要・フォルダ構成・コマンド一覧 |
> | `CLAUDE.md` | データ階層（Bronze/Silver/Gold）・探索戦略・公開範囲などの原則（AI 向け） |

## このリポジトリは何をする場所か（30秒）

議事録・顧客とのやり取り・共有資料・意思決定など、**案件のコンテキストを1か所に貯める器**です。人も AI（Claude Code）も、ここを参照して作業します。データは精製度で 3 層に捉えます（詳細は `CLAUDE.md`）。

- **Bronze（生記録）**: 文字起こし・課題のやり取り・資料原本・ソースコード。外部ツールの**同期ミラー**で、手編集しない
- **Silver（中間成果物）**: 議事録・ドキュメント・Wiki・変換済み資料
- **Gold（確定の判断材料）**: `Cortex/` 配下（Decisions・用語集・レポート・Home）。AI が最初に読む入口

仕組み（スキル・自動化）は中央の **cortex-engine** から配布されており、**このリポジトリでは初回に一度プラグインを入れるだけ**で使えます（リポジトリを Claude Code で開いてトラスト → インストール案内に「はい」）。

## こういう時はこうする（シナリオ早見表)

コマンド（スキル）は Claude Code で `/コマンド名` として実行します。全コマンドの一覧は `README.md` を参照。

| やりたいこと | こうする |
| --- | --- |
| この案件に新しく参加した（環境準備＋理解） | `/onboard-member`（プラグイン導入の案内＋Gold起点のオリエン） |
| 新しい定例・会議を自動取り込みしたい | **会議名の頭に案件キーを付け（例「【KC】定例」）、Google Meet の招待に cortex-notetaker の bot を追加**するだけ（文字起こしが自動で `会議/` に入り議事録も生成される）。※クラスメソッド側が主催する Google Meet のみ。顧客主催・Teams 等は `/post-meeting` で手動取り込み |
| 会議が終わった（議事録を作りたい） | `/post-meeting`（自動取り込み済み or 手元の文字起こしから議事録生成） |
| 顧客とのやり取り（課題）を最新化したい | **何もしなくてよい**（Webhook で数十秒＋平日毎時の自動同期）。手元を最新にするのは `/git-pull` |
| 課題に返信・更新したい | `/backlog-push`（編集してから実行すると Backlog に反映。個人の API キーが必要 → `/onboard-member` 参照） |
| もらった資料（PDF/PPTX/Excel等）を取り込みたい | `/sync-materials` |
| デザイン（画面一覧）を最新化したい | 毎晩自動同期。今すぐなら `/sync-designs` |
| Slack のやり取りを参照したい | `/read-chat`（`チャット/channels.json` のチャンネルを Slack MCP でライブ参照。リポには取り込まない・顧客には見えない） |
| 決まったこと（意思決定）を記録したい | 毎晩自動で追記される。今すぐなら `/update-decision-log` |
| 用語の定義を揃えたい | 毎晩自動で draft が追記される（**人間が確認して active に変える**）。手動なら `/update-glossary` |
| 最近の状況をざっと把握したい | `/catch-up-recent-status` |
| 週次の進捗をまとめたい | 毎週金曜に自動生成。手動なら `/weekly-report` |
| ソースコード・Wiki を手元で見たい | `/clone-dev-repos`（submodule として取得） |
| 変更を保存／最新化したい（非エンジニア向け） | `/git-save`（保存）・`/git-pull`（最新化）・`/git-fix-push`（push 失敗時の復旧） |
| Backlog/Figma/GitHub 以外のツールを使っている | `/customize-tooling`（別ツールで同等の仕組みを設計・実装） |
| Cortex（仕組み側）への要望・不具合がある | `/submit-feedback`（エンジンのリポジトリに Issue として登録） |
| 過去の経緯や仕様を AI に聞きたい | Claude Code で質問する（下記「AI との作業の仕方」） |

## どこに何を置くか

| ディレクトリ | 入れるもの | 編集してよい？ |
| --- | --- | --- |
| `課題管理/` | 課題管理ツール（Backlog 等）の課題・ドキュメント・Wiki | ❌ 同期ミラー。直さない（正本ツール側で直すと数十秒で反映される） |
| `会議/` | 会議の文字起こし・議事録 | 議事録は追記可。文字起こし原本は触らない |
| `チャット/` | Slack チャンネルのリンク（`channels.json`）。中身はミラーせず MCP でライブ参照 | `channels.json` にリンクを貼る |
| `共有資料/` | 提案書・外部 API 仕様などを Markdown 化したもの | `/sync-materials` 経由で更新 |
| `開発/` | ソースコード（submodule）・Wiki・GitHub Issues への道しるべ | ❌ submodule は正本リポで作業 |
| `デザイン/` | 画面インベントリ（Figma 同期）・スクショ等 | ❌ `inventory/` は自動同期。手編集しない |
| `Cortex/`（Gold層） | Decisions・用語集・レポート・Home | レビューはするが、原則スキル／自動更新が書く |

## AI との作業の仕方

Claude Code でこのリポジトリを開き、自然文で依頼・質問できます。AI は **まず `Cortex/`（Gold層）を読み、関係を辿って必要な生データだけを開く**という探索戦略で動きます（詳細は `CLAUDE.md`）。

例:

- 「クーポン機能の仕様の経緯を教えて」
- 「この課題に、前回の決定を踏まえて返信を下書きして」
- 「先週の動きを週次レポートにまとめて」

## やってはいけないこと

- **内部限定情報を書き込まない**: このリポジトリは顧客が生でアクセスする前提（AIS）です。売上・原価・工数・人事評価・率直なベンダー評価などは置かない（詳細は `CLAUDE.md` の「公開範囲」）
- **同期ミラーを手編集しない**: 上書きで消えます。正本側で直す
- **機密情報をコミットしない**: API キーは GitHub Secrets（自動化用）か、手元実行用なら環境変数に置く。保存場所は Claude Code の動作環境で変わる（CLI＝`.env`／デスクトップ＝ローカル環境エディタ／Web＝クラウド環境設定）
- **`Cortex/Home.md` に仕組みの話を書かない**: Home は Viewer の入口（顧客も読む面）。エンジン・プラグイン等への参照は書かない
