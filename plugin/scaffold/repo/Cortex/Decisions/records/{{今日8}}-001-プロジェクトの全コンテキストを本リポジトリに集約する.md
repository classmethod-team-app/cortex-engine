# プロジェクトの全コンテキストを本リポジトリに集約する

## 背景

プロジェクトのコンテキスト（MTGでの議論、顧客とのやり取り、共有資料、意思決定の経緯）は、議事録ツール・課題管理ツール・チャット・各人の記憶などに分散しがちである。分散したままでは、経緯の確認に時間がかかり、メンバーの入れ替わりで知識が失われ、AIエージェントに正確な文脈を与えることもできない。

## 理由

- コンテキストが1箇所に揃っていることが、人の意思決定とAIエージェントの出力品質の両方を支える土台になるため。
- 議事録・課題・資料・Decisionsを同一リポジトリに置くことで、git履歴がそのままプロジェクトの時系列記録になり、差分ベースの自動処理（Decision Log自動更新・週次レポート生成）が成立するため。
- 「ここを見れば全部ある」という正本を定めることで、経緯確認・引き継ぎ・横断分析のコストを恒久的に下げるため。
- 以降、コンテキストは発生の都度このリポジトリに取り込むことを原則とし、リポジトリ外にしか存在しない情報を作らない。
