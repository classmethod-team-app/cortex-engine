# Meetings（MTG議事録・文字起こし）

このディレクトリには、会議の文字起こしと議事録が格納されています。

## 文字起こしの自動取り込み（cortex-notetaker）

> **前提: クラスメソッド側が主催する Google Meet の会議のみ**が対象です。Gemini の文字起こしメモは主催者の Workspace で生成・共有されるため、**顧客側が主催する会議（先方の Google Meet / Teams / Zoom 等）は自動では入りません**。その場合は下の「議事録の手動生成」で取り込んでください。

対象の会議は、**運用は2つだけ**で文字起こしがこのディレクトリに自動で入り、議事録も自動生成されます（GitHub Actions `ingest-minutes`）。

1. **会議名の頭に案件キーを付ける**（例:「**【KC】定例**」。キーは `cortex-tools/infra/config.ts` の `key`）
2. **招待に `cortex-notetaker` の bot を追加する**（定例はシリーズに1回。招待されていれば bot の参加は不要）

- 取り込むか否かは「bot を招待したか」で決まります。**社内だけの会議には bot を招待しない**でください（このリポジトリは顧客に見える前提）
- どの案件リポへ入るかの判定は ①案件キー → ②`Cortex/Home.md` の client 名 → ③`ingest-config.json` の `meetingNamePatterns` の優先順。①②が会議名に入らない場合のみ③に**固有の会議名**を足します（「定例」のような汎用語は他案件を引き込むため入れない）
- この案件で自動取り込みを使うかは `ingest-config.json` の `enabled` で宣言します（`/setup-project` ステップ12）

## 議事録の手動生成

**顧客側が主催する会議（自動取り込みの対象外）**や、Teams・Zoom等の文字起こし、手元のファイルからは、文字起こしファイルをこのディレクトリに置いてスラッシュコマンドで生成します（`/post-meeting` でも可）。

```
/create-minute {文字起こしファイルのパス}
```

## ディレクトリ構成

```
会議/
├── Ph.1/                        # フェーズごと
│   └── YYYYMMDD/               # 日付ごと
│       ├── YYYYMMDD_minutes.md # 議事録（自動生成）
│       └── {元ファイル}.txt     # 文字起こし原本
├── templates/                   # 議事録テンプレート
│   └── minute-template.md
├── ingest-config.json           # 自動取り込みのルーティング設定（enabled / meetingNamePatterns）
└── README.md                    # このファイル
```

## 対応する文字起こし形式

| ツール          | ファイル形式    | 備考                                    |
| --------------- | --------------- | --------------------------------------- |
| Google Meet     | `.txt`          | Gemini によるメモ（タイムスタンプ付き） |
| Microsoft Teams | `.docx`         | 自動文字起こし                          |
| Zoom            | `.txt` / `.vtt` | 文字起こし                              |

## 議事録からのコンテキスト活用

生成された議事録は以下のワークフローで活用されます：

1. **Decision Log への反映**: `/update-decision-log` コマンドで決定事項を抽出
2. **課題管理ツールの更新**: 定例会のアジェンダ課題に結果を反映
3. **コンテキスト探索**: AIが作業開始時に関連する議事録を自動参照
