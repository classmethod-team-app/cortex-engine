現在のすべての変更を、GitHub（リモート）に保存します。非エンジニアでも使えるように、コミットメッセージはこちらで自動的に考え、`add` から `push` までを一括で実行します。

## このスキルがやること

1. 変更内容を確認する
2. 変更をすべてステージング（`git add`）する
3. 変更内容にふさわしい日本語のコミットメッセージを自動で作成して `commit` する
4. リモートに `push` する
5. push が分岐エラーで失敗した場合は、自動で取り込んで（rebase）からpushし直す

## 実行手順

1. `git status` と `git diff` で変更内容を確認する（何が変わったかを把握する）
2. 変更がない場合は「保存する変更はありません」と伝えて終了する
3. `git add -A` ですべての変更をステージングする
4. 変更内容を要約した**日本語の簡潔なコミットメッセージ**を作成する（命名規則は下記）
5. `git commit -m "<message>"` でコミットする
6. `git push` を実行する
7. push が **non-fast-forward（リモートと分岐）** で失敗した場合は、**`git-fix-push` スキルの手順**に従ってリモートを取り込んでから再push する
8. 完了したら、何を保存したか（コミットメッセージ）と結果をユーザーにやさしい言葉で報告する

## コミットメッセージの作り方

- 必ず**日本語**で、変更内容が一目で分かるように簡潔に書く
- 箇条書きは**使わない**（1行で書く）
- AIの署名は**追加しない**
- 例: 「定例の議事録を追加」「Decision Logを更新」「メンバー表を修正」

## 重要な制約

- 非エンジニア向けスキルのため、専門用語をそのまま報告に使わず、平易な言葉で結果を伝える
- 競合（コンフリクト）の解決で迷う場合は、勝手に判断せずユーザーに状況を説明して確認する
- `--force` などの強制pushは**行わない**
- 機密情報を含むファイル（`.env` など）が変更に含まれている場合は、コミット前に必ずユーザーに確認する
