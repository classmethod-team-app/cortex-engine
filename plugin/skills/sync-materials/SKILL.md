# Sync Materials

共有資料/ 配下の資料を Markdown に変換し、AI が読みやすい形で同期する。

**外部サービス（Google Drive 等）からの取得は行わない。** 資料は手元にダウンロードするか `共有資料/` に置いたうえで、**そのパスを渡して**変換する。

## 2 つの使い方

ユーザーの入力から、以下の 2 パターンを判別して処理する。

| #   | パターン       | 入力                                          | 用途                                             |
| --- | -------------- | --------------------------------------------- | ------------------------------------------------ |
| 1   | **取り込み**   | 手元/外部のファイルパス（例: `~/Downloads/会社紹介.pdf`） | `共有資料/` 配下にコピーして変換（元ファイルは残す） |
| 2   | **整理・変換** | `共有資料/` に置いたファイル or ディレクトリ  | その場で整理して変換（ディレクトリ指定で再帰一括） |

---

## スクリプトの場所と実行方法

変換スクリプトは**この Skill に同梱**されている（`scripts/convert.py`）。依存（markitdown 等）はスクリプト先頭の PEP 723 インラインメタデータで自己完結しており、`uv run --no-project` で隔離環境に自動解決される。**リポジトリの `pyproject.toml` には依存しない**ため、Skill ごと他リポジトリへコピーしても動く。

実行時は、Skill 起動時に提示される **「Base directory for this skill」** の絶対パスを `<SKILL_DIR>` として使う。`共有資料/...` のパスはリポジトリルート（カレントディレクトリ）基準。

```bash
uv run --no-project "<SKILL_DIR>/scripts/convert.py" <引数...>
```

---

### パターン 1：取り込み（手元のファイル → 共有資料/）

手元（ダウンロードフォルダ等）にあるファイルを `共有資料/` 配下に取り込みつつ変換する。`--dest` に配置先の親ディレクトリ、`--name` に簡潔な名前を渡す。

```bash
uv run --no-project "<SKILL_DIR>/scripts/convert.py" "~/Downloads/会社紹介資料.pdf" --dest "共有資料/" --name "クラスメソッド会社紹介資料"
```

これで `共有資料/クラスメソッド会社紹介資料/{クラスメソッド会社紹介資料.pdf, .md}` が作られる（元ファイルはコピーなので手元にも残る）。`--name` を省略すると元ファイル名がそのまま使われる。

**命名ガイド（`--name` に渡す名前）**

- 資料の中身を端的に表す名前にする（フォルダ名＝ファイル名＝参照ID `material:{slug}` の元になる）
- 次のようなノイズは落とす: 用途・配布先の接頭辞（`〇〇用_`）、日付・版番号（`_v2`・`20260617`）、`コピー`・`最新版`・`(1)` 等
- 拡張子は付けない（スクリプトが補う）。スペースより中黒・アンダースコアを使い、パスとして扱いやすくする

### パターン 2：整理・変換（共有資料/ に置いたものを変換）

`共有資料/` に直接置いたファイルを、その場で整理（同名ディレクトリ作成＋移動）して変換する。

```bash
uv run --no-project "<SKILL_DIR>/scripts/convert.py" "共有資料/会社紹介資料.pdf" --organize
```

親ディレクトリを指定すると、配下を**再帰的に**走査し、未整理の対応ファイルをまとめて変換する。

```bash
uv run --no-project "<SKILL_DIR>/scripts/convert.py" "共有資料/" --organize
```

---

### 変換方式：markitdown のみ（AWS/Bedrock 不要）

`.xlsx` / `.docx` / `.pptx` / `.pdf` / `.txt` / `.csv` 等のテキスト抽出可能な形式は、スクリプトが **markitdown で自動変換**する（安い・速い・AWS 認証不要）。Claude や Bedrock による変換は行わない。

**画像（`.png` / `.jpg` / `.jpeg` / `.gif` / `.webp` / `.bmp` / `.tiff` / `.svg`）は変換せず、生ファイルのまま置く。** 図やスクリーンショットの内容が必要になったときは、Claude が生の画像ファイルを直接読めばよい（`.md` は作らない）。

元ファイル（Bronze）は編集・削除せず残す。frontmatter は付けない（frontmatter を持つのは Gold 層のみ＝オントロジー規約。参照 ID `material:{ファイル名（拡張子なし）}` はファイル名から機械的に導出される）。

## 補足：スクリプトの自動処理

- **取り込み（--dest）**: 手元のファイルを `<親dir>/<--name または元ファイル名のstem>/` にコピー → markitdown で変換。元ファイルは手元にも残る。
- **整理・変換（--organize）**: 拡張子を除いた名前のディレクトリを作成 → 元ファイルを移動 → markitdown で変換 → `.md` を保存。ディレクトリ指定時は配下の未整理ファイルを再帰的に処理。既に整理済みフォルダ内（`{名}/{名}.ext`）のファイルは再移動せずその場で `.md` を生成する。既に `.md` があるものはスキップ（べき等）。
- **単発変換（-o / 引数のみ）**: 対応形式を md にして指定パスへ保存、または標準出力。
- **画像**: 変換対象外。生のまま置く（ログに「画像は変換対象外」と出力し、`.md` は作らない）。

```
共有資料/
  提案書/                  # 配置先フォルダ（直下にファイルを置く）
    ○○提案書.md           # 変換後Markdown
    ○○提案書.pptx         # 実データ（残す）
  外部API/
    △△API仕様.md
    △△API仕様.pdf
```

## 対応形式

- `markitdown`（Microsoft 製）で自動変換（AWS/Bedrock 不要）: `.xlsx`, `.docx`, `.pptx`, `.pdf`, `.txt`, `.csv`, `.html`, `.htm`, `.json`, `.xml`, `.rtf`
- **変換しない（生のまま置く）**: 画像（`.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.bmp`, `.tiff`, `.svg`）。必要時に Claude が生ファイルを直接読む
