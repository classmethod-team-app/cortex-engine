このテンプレート（Cortex）を複製して使う中で気づいた**改善要望・不具合・質問・新スキル要望**を、テンプレートの提供元リポジトリに Issue として登録します。複製先の案件リポジトリから実行しても、フィードバックは常に**テンプレートの upstream リポジトリ**に集約されます（複製先の案件リポジトリには作りません）。

> **フィードバックの送り先（upstream）**: `classmethod-internal/aidd-project-cortex`
> テンプレートが別組織（例: `classmethod-internal`）へ移管された場合は、この値を更新してください。

## 前提

- `gh`（GitHub CLI）が認証済みで、送り先リポジトリに Issue を作成できる権限があること（`gh auth status` で確認）。未認証・権限不足なら `gh auth login` や権限確認を案内する。

## 手順

### 1. フィードバック内容のヒアリング

- **種別**: 不具合 / 改善提案 / 新スキル・機能要望 / 質問 / その他
- **タイトル**（簡潔に）
- **詳細**（何が・どうなってほしいか。不具合なら再現手順・期待と実際）
- 任意: 関連するスキル名・ファイル・ステップ

### 2. Issue 本文の組み立て

以下のテンプレートで本文を作成する（種別に応じて項目は適宜調整）。

```markdown
## 種別

<不具合 / 改善提案 / 機能要望 / 質問 / その他>

## 内容

<本文>

## 再現手順（不具合の場合）

1.
2.

## 期待する挙動 / あるべき姿

<...>

## 補足

- 複製元テンプレート: aidd-project-cortex
- 報告日: <YYYY-MM-DD>
```

> **機密情報を含めない**: 顧客名・具体的なやり取り・認証情報など、案件の機密はフィードバックに書かない。送り先のテンプレートリポジトリは案件リポジトリとは別の公開範囲です。再現に必要なら、機密を伏せた最小例にする。

### 3. 内容の確認（外部書き込みの承認）

作成する Issue（送り先リポジトリ・タイトル・本文）をユーザーに提示し、**🚨 で目立たせて明示的に承認を得てから**作成する。

```
🚨🚨🚨 以下の内容で classmethod-internal/aidd-project-cortex に Issue を作成します。よろしいですか？ 🚨🚨🚨
（タイトルと本文を表示）
```

### 4. Issue の作成

承認後、`gh` で作成する。

```bash
gh issue create \
  --repo classmethod-internal/aidd-project-cortex \
  --title "<タイトル>" \
  --body "<本文>"
```

- ラベルを付ける場合は、`gh label list --repo classmethod-internal/aidd-project-cortex` で**存在を確認できたものだけ** `--label` に指定する（存在しないラベルはエラーになる）。
- 作成後、発行された Issue の URL をユーザーに伝える。

## 注意事項

- 送り先は常にテンプレートの upstream（複製先の案件リポジトリではない）
- 機密情報を含めない（上記）
- 外部リポジトリへの書き込みなので、必ずステップ3の承認を経てから作成する
