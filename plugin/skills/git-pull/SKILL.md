---
GitHub（リモート）にある**最新の状態**を、手元（ローカル）に取り込みます。他のメンバーが更新した内容を反映したいときに使います。非エンジニアでも安全に使えるように、手元の変更を失わないよう配慮して実行します。

## このスキルがやること

1. 手元に未保存（未コミット）の変更がないか確認する
2. リモートの最新を取得する
3. 履歴をきれいに保つ形（rebase）で最新状態を取り込む
4. 結果をやさしい言葉で報告する

## 実行手順

1. `git status -sb` で、手元に未コミットの変更があるか・リモートとどれだけ差があるかを確認する
2. **未コミットの変更がある場合**:
   - 勝手にコミットや破棄をせず、ユーザーに「手元に保存していない変更があります」と伝える
   - 先に保存したいか（`git-save` スキル）、いったん退避したいか（`git stash`）を確認してから進める
3. `git fetch origin` でリモートの最新を取得する
4. `git pull --rebase origin <現在のブランチ>` で最新状態を取り込む
5. 競合（コンフリクト）が発生した場合は、**`git-fix-push` スキルの「競合解決ルール」**に従って解決し、`git rebase --continue` で続行する
6. `grep -rn "<<<<<<<\|>>>>>>>\|=======" .` で競合マーカーの消し残しがないことを確認する
7. 完了したら「最新の状態にしました」と、新しく取り込まれた主な変更をやさしく報告する

## 重要な制約

- 非エンジニア向けスキルのため、専門用語をそのまま使わず平易な言葉で報告する
- 手元の未保存の変更を、ユーザーの確認なしに**破棄しない**
- マージコミットを増やさないよう、`git pull` ではなく **`git pull --rebase`** を使う
- 競合解決で迷う場合（自動生成でないドキュメントや課題本文など）は、勝手に判断せずユーザーに状況を説明して確認する
- うまくいかない場合は `git rebase --abort` で元の状態に戻せることを念頭に置く
