**既にセットアップ済みの案件コンテキストリポジトリに、新メンバーが参加してクローンした直後**に使うスキルです。(1) 参加者個人のローカル環境を整え、(2) 案件を理解する、までを支援します。

> `setup-project` との違い: `setup-project` は**リポジトリを新規作成する人**向け（プレースホルダ置換・リポジトリのSecrets設定など、1回だけ・全員に共通の準備）。本スキルは**既存リポに参加する人**向けで、リポ自体の準備は完了済みである前提。リポの準備状況は `/setup-status` で確認できる。

## パート1: ローカル環境の準備

上から順に確認・案内する（各ステップは実行前にユーザーへ確認）。

1. **ツール**: `mise install` で node/pnpm/gh/jq を揃える。`pnpm --version` / `gh --version` / `jq --version` で確認
2. **GitHub 認証**: `gh auth status`。未認証なら `gh auth login`
3. **ルールファイル生成**: `pnpm dlx rulesync@latest generate`（`.claude/` 等をローカルに展開し、スキルを有効化）
4. **開発リポジトリ（submodule）**: 開発リポがある案件なら `git submodule update --init`（または `/clone-dev-repos`）
5. **認証情報（任意・ローカルで手動同期する場合のみ）**:
   - 日々の同期は GitHub Actions の cron が自動実行するため、**通常は `git pull` で最新を取得できる**（手元の認証情報は必須ではない）
   - 自分のマシンで `/backlog-pull`・`/sync-designs` を**手動実行したい場合のみ**、個人の認証情報を環境変数で設定する
   - 例: `DOMAIN`（共有値）/ `PROJECT_KEY`（共有値）/ `BACKLOG_API_KEY`（**各自の**APIキー）/ `FIGMA_TOKEN`（各自・任意）
   - **保存場所は使っている Claude Code の動作環境で変わる**: ターミナルCLIなら `.env`（`.gitignore`済み）、デスクトップアプリなら**ローカル環境エディタ**（暗号化）、Web版なら**クラウド環境設定の環境変数**。詳細・注意（Webは編集者に可視など）は `credentials` ルール参照
   - **秘密情報はリポジトリにコミットしない**
6. **ハーネスプラグイン**: 案件で使うハーネス（PM/開発/デザイン等）を Claude Code に導入する（`/plugin marketplace add ...` → `/plugin install ...`）。案件固有値は `.claude/settings.json` の `pluginConfigs`
7. **（任意）日本語ディレクトリの英語別名**: ターミナルで日本語ディレクトリ（`会議/` `共有資料/` 等）への移動が面倒な場合は、`sh "<SKILL_DIR>/scripts/setup-aliases.sh"` を実行すると英語別名（`Meetings` `Materials` 等）の symlink が張れる。**ローカルのみ・コミットされない**（正本は日本語のまま）。非エンジニアや別名が不要な人は実行しなくてよい。

## パート2: プロジェクトの理解（Gold起点のオリエンブリーフ）

`CLAUDE.md` の探索戦略（**Gold起点の二段構え**）に従い、生データの全文検索から始めず、まず `Cortex/`（Gold層）を読む。以下を読んで新メンバー向けのオリエンブリーフを作成する。

1. `Cortex/Home.md` — 案件の全体像・識別カード（kind / client / domains / platforms）・使用ツール
2. `Cortex/用語集/` — 案件固有の語彙（最初に揃えて表記揺れ・誤解釈を防ぐ）
3. `Cortex/Decisions/` — 主要な確定事項（特に現行版。`supersedes` で置き換えられた古い決定は現行で上書きされている点に注意）
4. `Cortex/レポート/` — 直近の動き・進行中の論点
5. `.rulesync/rules/member.md` — 体制・関係者（あれば）

### 出力するオリエンブリーフ

- **この案件は何か**: 目的・クライアント・業務ドメイン・プラットフォーム
- **体制と関係者**: 誰が何の役割か
- **直近の状況と進行中の論点**: （詳細は `/catch-up-recent-status` に委譲してよい）
- **押さえるべき主要な決定事項**（現行版）
- **重要な用語**（最小セット）
- **どこに何があるか**: `USAGE.md`（日々の使い方）・`README.md`（構成・コマンド一覧）へのポインタ
- **次の一歩**: よく使うコマンド（`/catch-up-recent-status`・`/post-meeting` 等）

## 注意事項

- 個人の認証情報は `.env` / 各自の環境に置き、**コミットしない**
- リポジトリ自体が未セットアップ（プレースホルダ `{{ }}` が残っている等）なら、まず `/setup-status`・`/setup-project` を案内する
- 理解パートは必ず **Gold起点**（リポジトリ全体の全文検索から始めない）
