description: 定例会後の処理を一気通貫で実行する。会議の文字起こし（自動取り込み済み or 手動で渡されたもの）から議事録を生成する
---
会議の文字起こしから議事録生成までを一気通貫で実行します。`create-minute`（議事録生成）への薄いオーケストレーションです。各処理の詳細は `create-minute` の規約に従い、ここでは重複定義しません。

`$ARGUMENTS` には、手元の文字起こしファイルのパス、または会議日（`YYYY/MM/DD`）を任意で渡せます。

## 文字起こしの入手方法

文字起こしは次のどちらかで用意します。

- **自動取り込み（Google Meet）** — `cortex-notetaker` Bot が招待された会議の文字起こしを取り込み、`会議/` に配置します（`会議/ingest-config.json` で有効化した案件）。この場合、文字起こしは**既にリポジトリに入っている**ので、取得は不要です。
- **手動** — Google Meet 以外（Teams 等）や自動取り込み未設定の案件では、文字起こしファイル（`.txt` / `.docx` / プレーンテキスト）を手元に用意し、そのパスを `$ARGUMENTS` で渡します。

## 手順

1. **文字起こしを特定・配置する**
   - `$ARGUMENTS` にファイルパスが渡された／添付された → 内容や名前から会議日（`YYYYMMDD`）を特定し、`会議/Ph.1/YYYYMMDD/` を作成して元ファイルをそこへコピーする。日付が判別できなければユーザーに確認する。
   - パス指定が無い → `会議/` 配下から、**まだ議事録が無い文字起こし**（自動取り込み済みのもの）を特定する。複数あればユーザーに対象を確認する。

2. **議事録を生成する**
   - `create-minute` スキルの手順に従い、同じディレクトリに `YYYYMMDD_minutes.md` を生成する（frontmatter・テンプレート・記載ルール・話者の敬称等はすべて `create-minute` の規約に従う）。
   - `create-minute` は冒頭で `catch-up-recent-status` による直近状況（定例・課題・Decision Log）の把握と、Gold層（`Cortex/Home.md`・用語集・Decisions）・メンバー表の読み込みによる用語/人名の正規化を行ってから議事録化するため、**プロジェクトを理解した精度の高い議事録**になります（その過程は `create-minute` 側に組み込み済み）。

3. **完了報告**
   - 文字起こしの配置先と、生成した議事録のパスを報告する。
   - 補足として、生成した議事録は夜間の Decision Log / 用語集 の自動更新が git 差分として自動的に拾うため、**このスキルでは決定抽出までは行わない**ことを伝える。

## 補足

- 文字起こしの自動取得は本スキルの外（`cortex-notetaker` Bot ＋ Apps Script ＋ `ingest-minutes` ワークフロー）で完結します。本スキルは**手元の文字起こし、または取り込み済みの文字起こし**から議事録化するところを担います。
- 決定の記録・用語の登録は、それぞれ `update-decision-log` / `update-glossary`（または夜間の自動更新）に任せます。
