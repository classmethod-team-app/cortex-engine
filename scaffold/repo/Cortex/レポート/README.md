# 週次レポート

単一案件の「その週に何が起きたか」をまとめた週次レポートを格納する。

- 生成: `weekly-report` スキル（手動 `/weekly-report` または週次cron `.github/workflows/weekly-report.yml`）
- 配置・ファイル名: `records/YYYYMMDD.md`（`YYYYMMDD` は対象週の末日＝生成日。1週1ファイル・再実行で上書き）

## なぜフォーマットを固定するか

このリポジトリは案件ごとに複製される。**全案件が同一フォーマットの週次レポートを持つ**ことで、巡回エージェントが複数案件を横断して機械的に読み取り・集約できる。よって frontmatter と見出しの**並び・文言は固定**とし、案件ごとに独自項目を足さない（該当が無いセクションは削除せず「なし」と記す）。

## 標準フォーマット

雛形は [`template.md`](./template.md)（**これが正本**。frontmatter・見出しともこの構造から変更しない）。

### frontmatterフィールド

| フィールド | 説明 |
| --- | --- |
| type / id | オントロジー規約参照（`id` の `YYYYMMDD` は期間末日） |
| project | プロジェクト名 |
| period_start / period_end | 対象期間（末日の6日前〜末日＝生成日） |
| generated_at | 生成日 |
| metrics | 更新課題数・新規議事録数・新規Decisions数・コミット数・マージPR数 |

### 見出し（この6つ・この順で固定）

| 見出し | 内容 |
| --- | --- |
| 今週の主な動き | 3〜5行のサマリ |
| 決定事項 | その週に確定した Decisions の差分（無ければ「なし」） |
| 進行中・動きのあった課題 | 新規・更新・回答のあった課題 |
| リスク・懸念・ブロッカー | 停滞・未回答・障害など（無ければ「特になし」） |
| 来週の予定・次アクション | 次にやること・締切・依頼待ち |
| 数値サマリ | metrics と対応する件数 |

## 対象期間

直近7日（生成日を末日とした7日間）。詳細は `weekly-report` スキル参照。
